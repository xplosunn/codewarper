Codewarper

The coding agent you can standardize on.

Most coding agents come with tools built in and hope you'll lock them down. Codewarper does the opposite. It only can do what you explicitly configure.

Configured once. Used consistently.

The point is not per-user customization. The point is an agent your organization can shape to its own requirements, then adopt as a standard.

No tools by default

Ships with zero capabilities. File access, shell commands, network calls — nothing is available until you add it to the config.

Zero dependencies

No runtime packages. No supply chain to trust beyond Node.js itself.

No telemetry

Never phones home. No usage tracking, no crash reports. The only outbound connections are the provider APIs you configure.

Auditable code

Full source on GitHub. Review it, fork it, ship it internally.

Let the AI write your config

On first run, Codewarper asks which provider you want to use, then has that model interview you and generate a codewarper.js matched to your project. Edit the result, check it in, done. 

export default {
  tools: [ /* file ops, grep, web search, ... */ ],
  commands: {
    test: { description: "Run the test suite", async run() { /* ... */ } },
  },
  systemPrompt: "You are a concise coding assistant. ...",
  hooks: {
    onProviderRequest(req) { /* log it */ },
    onProviderResponse(req, res) { /* audit it */ },
  },
}

Custom slash commands

Define your own /commands in the config. Run tests, open your editor, trigger CI — anything you can script, the agent can invoke.

Get started

Install codewarper from npm.

npm install -g codewarper

Run it.

codewarper
